A hardware implementation of density-field obstacle avoidance for a Unitree Go1 navigating around moving pedestrians using onboard stereo perception and high-level velocity commands.

Robot System

The current system uses a ROS Noetic perception container on the Jetson AGX Xavier and a Go1 control node that publishes high-level forward-speed and yaw-rate commands.

1. ZED2i perception

The camera node uses the ZED SDK Python API for object detection, tracking, and pose. It publishes human detections as autoware_msgs/DetectedObjectArray on /detections.

2. Density planning

The experiment node converts detections into world-frame moving obstacles, computes a density-field control direction, and records synchronized marker and velocity bags for analysis.

3. Go1 command tracking

The planner publishes Unitree HighCmd messages to /high_cmd. The lower-level robot stack executes the high-level velocity command on the physical quadruped.

Hardware Stack

The robot carries a ZED2i stereo camera, Jetson AGX Xavier, external MAXOAK K2 battery, and a custom 3D-printed camera/battery mounting setup. The Jetson runs JetPack 5.1.2 / L4T R35.4.1 with ZED SDK 5.1.2 and ROS Noetic inside the perception container.

Reproduce

The repository supports two Jetson container paths: load the private saved lab image for exact recovery, or rebuild from the public Stereolabs ZED Jetson base image and the checked-in Jetson nodes.

Exact lab image restore

make docker-load
make docker-run

Clean rebuild from source

make docker-build
make docker-run

Run the Go1 experiment

roscore
python3 ros_nodes/motion_plan_exp.py

Analyze saved bags

scripts/sync_jetson_bags.sh
/usr/bin/python3 scripts/analyze_bag_runs.py
